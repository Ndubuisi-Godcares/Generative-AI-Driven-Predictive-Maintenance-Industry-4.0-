# Generative AI-Driven Predictive Maintenance for Industry 4.0

## Overview

This project applies a Wasserstein Conditional Generative Adversarial Network (WCGAN) to synthesize industrial machine failure data, and then trains predictive maintenance classifiers on both real and synthetic data. The goal is to predict machine failures one day in advance, addressing the critical challenge of data scarcity in industrial settings where failure events are rare.

The pipeline covers the full machine learning lifecycle: data ingestion, exploratory analysis, missing value imputation, feature engineering, synthetic data generation, quality evaluation, and failure prediction using XGBoost and Convolutional Neural Networks.

---

## Problem Statement

Predictive maintenance in Industry 4.0 relies on sufficient labeled failure data, which is often unavailable due to the infrequent nature of machine faults. This project uses generative modeling to augment real sensor data with high-fidelity synthetic samples, enabling more robust training of downstream predictive models.

## Dataset

- **Source:** Kaggle - `binaicrai/machine-failure-data`
- **File:** `machine_failure_data.csv`

- **Features:** Temperature readings (min/max), leakage, evaporation, electricity consumption, multiple operational parameters recorded at 9am and 3pm, and failure indicators for the current day.

## Pipeline

### 1. Data Ingestion
Data is downloaded from Kaggle, extracted, and loaded as a Pandas DataFrame with date parsing.

### 2. Exploratory Data Analysis (EDA)
- Distribution plots, box plots, and pair plots for numerical features
- Correlation heatmap (Pearson)
- Time-series visualization of electricity consumption colored by failure status
- Statistical moments: skewness and kurtosis computed per feature to guide imputation strategy

### 3. Missing Value Imputation
Missing values are handled using Iterative Imputation (MICE) via scikit-learn's `IterativeImputer`, with categorical columns one-hot encoded prior to imputation. A Savitzky-Golay filter is then applied to smooth the numerical columns.

### 4. Feature Engineering
Additional condition-based features are derived from the imputed data:

- Vibration/speed proxy (absolute first difference of speed parameter)
- Mean, variance, and skewness of temperature and leakage
- Entropy of temperature and electricity (rolling window)
- Cycle count from morning vs. afternoon parameter comparisons
- Temperature gradient and electricity load proxy

### 5. Preprocessing
- Min-Max scaling to the range `[-1, 1]` applied to both the imputed data and the condition features before feeding into the WCGAN.

### 6. Synthetic Data Generation (WCGAN)

A Wasserstein Conditional GAN with Gradient Penalty (WGAN-GP) is implemented in PyTorch.

**Generator architecture:**
- Input: noise vector (dim=50) concatenated with condition vector
- Four fully connected layers with ReLU activations
- Output layer with Tanh activation (normalized to `[-1, 1]`)

**Discriminator architecture:**
- Input: data concatenated with condition vector
- Five fully connected layers with LeakyReLU and Dropout
- Scalar output (no sigmoid; Wasserstein loss)

**Training details:**
- Wasserstein loss with gradient penalty (lambda = 10)
- Adam optimizer, learning rate = 1e-4
- 5 discriminator steps per generator step
- 150 epochs, batch size = 64
- Early stopping based on FID score (patience = 10)
- GPU-accelerated (CUDA)

**Synthetic data scale:** 500,000 samples generated from the trained generator.

### 7. Synthetic Data Evaluation

Quality of synthetic data is assessed using:

- **Frechet Inception Distance (FID):** Measures distributional similarity via mean and covariance of features
- **Wasserstein Distance:** Per-feature and overall distribution distance
- **KL Divergence:** Discretized, per-feature and combined
- **Maximum Mean Discrepancy (MMD):** GPU-accelerated via CuPy with RBF kernel
- **Statistical moment comparison:** Mean, variance, skewness, and kurtosis differences
- **Anderson-Darling and Kolmogorov-Smirnov tests:** Per-feature normality and distributional tests
- **PCA visualization:** 3D scatter plots of top and bottom principal components
- **t-SNE visualization:** 2D and 3D, with multiple perplexity values explored
- **KDE and histogram overlays:** Per-feature density comparison

Reported evaluation scores on the test set:

| Metric              | Score  |
|---------------------|--------|
| Wasserstein Distance | 0.1128 |
| FID Score            | 0.0199 |
| MMD Score            | 0.0210 |

### 8. Predictive Maintenance Classification

The target is `Fail_tomorrow_Yes`. Features `Failure_today_Yes`, `Failure_today_No`, `Fail_tomorrow_No`, and `Location` are dropped before modeling.

Three training scenarios are evaluated:

- **Real data only** (original imputed dataset)
- **Synthetic data only** (WCGAN-generated)
- **Combined dataset** (real + synthetic)

For each scenario, two scaling strategies are compared:

- **Min-Max Scaling** (range `[0, 1]`)
- **MaxAbs Scaling**

**Models:**

**XGBoost** with Optuna hyperparameter tuning (50 trials, TPE sampler, 5-fold cross-validation):
- Parameters tuned: `n_estimators`, `max_depth`, `learning_rate`, `subsample`
- GPU-accelerated with `tree_method='hist'` and `device='cuda'`

Best XGBoost results on real data (Min-Max scaling):

| Metric    | Score  |
|-----------|--------|
| Accuracy  | 0.9683 |
| Precision | 0.9755 |
| Recall    | 0.9516 |
| F1-Score  | 0.9634 |
| ROC-AUC   | 0.9953 |
| Log Loss  | 0.0853 |
| RMM       | 0.0484 |

**CNN (1D Convolutional Neural Network)** with Optuna hyperparameter tuning (50 trials):
- Parameters tuned: `num_filters`, `kernel_size`, `dense_units`, `batch_size`, `epochs`
- Architecture: Conv1D -> MaxPooling1D -> Flatten -> Dense -> Sigmoid output
- Early stopping on validation loss (patience = 5)
- Evaluated on both real and synthetic data

### 9. Model Explainability

SHAP (SHapley Additive exPlanations) is used to interpret XGBoost predictions:
- `TreeExplainer` for global feature importance (summary plots)
- Force plots for individual predictions
- Results saved as interactive HTML files

### 10. Additional Evaluation Metrics

- Precision-Recall curve and AUC-PR
- Average Precision Score
- Matthews Correlation Coefficient (MCC)
- Calibration curve and Brier Score
- Lift curve
- Cumulative Gain Chart
- Profit Curve (with configurable false-positive and false-negative costs)
- Estimated downtime reduction based on recall
- F-beta score (beta=2)
- Prediction latency measurement

## Key Design Decisions

- **WGAN-GP over standard GAN:** Wasserstein loss with gradient penalty provides more stable training and a meaningful loss metric that correlates with generation quality, avoiding mode collapse.
- **Conditional generation:** Conditions derived from engineered features (temperature gradient, entropy, cycle count, load proxy) guide the generator toward physically meaningful synthetic samples.
- **Iterative imputation:** MICE-based imputation is preferred over simple mean/median fill to respect multivariate dependencies in sensor data.
- **Dual scaling experiments:** Both Min-Max and MaxAbs scaling are tested across all model configurations to assess sensitivity of results to the normalization choice.
- **Three-way data evaluation:** Training on real-only, synthetic-only, and combined datasets isolates the contribution of synthetic augmentation to predictive performance.

## Limitations

- The notebook is designed for Google Colab and uses Colab-specific APIs (`google.colab.files`, `drive.mount`). Adaptation is needed to run locally.
- CuPy-based MMD computation requires a compatible CUDA installation.
- The `cuml` library (RAPIDS) is referenced but optional; standard scikit-learn equivalents are used throughout the main pipeline.
- Synthetic data quality is sensitive to the number of WCGAN training epochs and the chosen noise dimension.
